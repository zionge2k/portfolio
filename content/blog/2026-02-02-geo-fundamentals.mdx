1~3편에서 전통적인 SEO를 다뤘다. 이제 AI 검색 시대의 새로운 최적화, **GEO**를 다룬다.

## GEO란

**Generative Engine Optimization** — AI 검색엔진의 답변에 내 콘텐츠가 **인용**되도록 최적화하는 것이다.

SEO와 비교하면:

| | SEO | GEO |
|--|-----|-----|
| **목표** | 검색 결과 상위 노출 | AI 답변에 인용 |
| **결과 형태** | 10개의 파란 링크 | 하나의 합성된 답변 + 2~7개 출처 |
| **성공 지표** | 순위, 클릭률, 트래픽 | 인용 횟수, AI 가시성 점수 |
| **콘텐츠 형태** | 키워드 최적화 페이지 | 구조화되고, 인용 가능한, 데이터 중심 콘텐츠 |

한 마디로: **SEO는 "검색되는 것", GEO는 "인용되는 것"**이다.

## 왜 GEO가 중요한가

숫자로 보면:

- Google 검색의 **25%** 이상이 AI Overview를 표시한다 (2025 기준)
- AI Overview가 표시되면 기존 상위 콘텐츠의 클릭률이 **34.5% 감소**한다
- Google 검색의 **60%** 가 제로클릭(사용자가 링크를 클릭하지 않음)이다
- Gartner는 2028년까지 전통적 검색 트래픽이 **50% 감소**할 것으로 예측한다
- Z세대의 **35%** 가 AI 도구를 첫 번째 검색 수단으로 사용한다

사용자가 Google에 질문하면, AI가 답변을 합성해서 보여주고, 사용자는 그 답변만 읽고 떠난다. 내 사이트에 방문하지 않는다. **AI의 답변에 인용되는 것이 새로운 1페이지 랭킹**이다.

## AI 검색엔진의 동작 원리: RAG

AI 검색엔진은 **RAG (Retrieval-Augmented Generation)** 파이프라인으로 동작한다.

```
사용자 질문
    ↓
1. 웹 검색 (관련 문서 검색)
    ↓
2. 문서 청크 추출 (가장 관련 있는 부분만)
    ↓
3. LLM에 전달 (검색 결과를 컨텍스트로)
    ↓
4. 답변 생성 + 출처 표기
```

전통적 검색엔진과의 핵심 차이:
- Google은 **링크 목록**을 보여주고 사용자가 직접 읽는다
- AI 검색엔진은 **여러 문서를 읽고 합성한 답변**을 보여준다
- 출처는 인라인 인용(`[1]`, `[2]`)으로 표기된다

## 플랫폼별 차이

모든 AI 검색엔진이 같은 방식으로 인용하지 않는다. 플랫폼별로 선호하는 출처가 다르다.

### Perplexity

가장 검색 중심적인 플랫폼이다. 모든 질문에 실시간 웹 검색을 수행한다.

- 2000억개 이상의 URL 인덱스에서 검색
- **문서 단위가 아닌 청크(문단) 단위**로 관련 부분을 추출
- 가장 많이 인용하는 출처: **Reddit** (6.6%)
- "검색하지 않은 내용은 답변에 포함하지 않는다"는 원칙

### ChatGPT

두 가지 모드로 동작한다.

- **웹 검색 없이**: 학습 데이터만으로 답변. 출처 링크 없음. 환각 가능
- **웹 검색과 함께**: RAG 방식으로 동작. 출처 인용
- 가장 많이 인용하는 출처: **Wikipedia** (7.8%)
- 전체 AI 검색 트래픽의 **87.4%** 를 차지

### Google AI Overview

기존 Google 검색 인프라 + Gemini 모델 조합이다.

- 하나의 질문에 **여러 하위 검색**을 실행하고 결과를 합성
- 인용의 **93.67%** 가 기존 검색 상위 10위 내 사이트에서 나옴
- 하지만 그 중 **47%** 는 5위 이하의 페이지에서 인용
- **E-E-A-T(경험, 전문성, 권위, 신뢰)** 가 가장 강력한 필터. 인용의 96%가 E-E-A-T가 강한 출처
- 응답당 평균 **4개 도메인**에서 **10.2개 링크** 인용

### 핵심: 플랫폼 간 겹침이 적다

ChatGPT와 Perplexity **모두**에서 인용되는 도메인은 **11%** 에 불과하다. 하나의 플랫폼에만 최적화하면 안 된다.

## 연구로 검증된 9가지 전략

2023년 Princeton, Georgia Tech, IIT Delhi 연구진이 **"GEO: Generative Engine Optimization"** 논문을 발표했다 (KDD 2024 정식 발표). 10,000개 쿼리로 9가지 최적화 전략을 테스트한 결과다.

### 효과가 큰 전략 (30~40% 가시성 향상)

**1. 출처 인용 (Cite Sources)**

주장에 신뢰할 수 있는 출처를 인용한다.

```markdown
<!-- Bad -->
React는 빠른 UI 라이브러리이다.

<!-- Good -->
React는 가상 DOM을 사용하여 렌더링 성능을 최적화한다
(React 공식 문서, https://react.dev).
```

효과: 하위 5위 사이트에서 **115.1% 가시성 증가**. GEO의 가장 강력한 전략이다. 특히 사실 확인이 중요한 질문(factual query)에서 효과적이다.

**2. 통계 추가 (Statistics Addition)**

정성적 설명을 정량적 데이터로 대체한다.

```markdown
<!-- Bad -->
Next.js는 많은 기업이 사용한다.

<!-- Good -->
Next.js는 Fortune 500 기업 중 25% 이상이 사용하며,
npm 주간 다운로드 수가 700만 회를 넘는다 (2025 기준).
```

효과: 평균 **28~40% 가시성 향상**. Perplexity에서는 **37%** 향상으로 가장 높은 효과.

**3. 인용문 추가 (Quotation Addition)**

관련 전문가의 직접 인용을 포함한다.

```markdown
Dan Abramov는 "서버 컴포넌트는 번들 사이즈를
근본적으로 줄이는 방법"이라고 설명했다.
```

효과: Perplexity에서 **22% 향상**. 인물, 사회, 역사 관련 주제에서 가장 효과적.

### 보통 효과의 전략 (15~30% 향상)

**4. 가독성 최적화 (Fluency Optimization)**

읽기 쉽고 자연스러운 문장으로 다듬는다. AI는 정보의 내용뿐 아니라 **표현 방식**도 평가한다.

- 통계 추가와 결합하면 단일 전략 대비 **5.5% 추가 향상** (최고 조합)
- 출처 인용과 결합하면 평균 **31.4% 향상**

**5. 쉬운 설명 (Easy-to-Understand)**

복잡한 개념을 단순하게 설명한다. 문장은 15~20단어, 문단은 60~100단어가 적당하다.

### 효과가 없거나 역효과인 전략

**6. 키워드 채우기 (Keyword Stuffing)** — 전통 SEO의 대표 기법이지만, AI 검색에서는 **10% 악화**. 오히려 역효과.

**7. 권위적 어조 (Authoritative Tone)** — 설득적으로 쓴다고 인용이 늘지 않는다. AI는 어조보다 내용을 본다.

## SEO와 GEO의 관계

GEO는 SEO를 대체하는 것이 아니다. **SEO 위에 쌓는 것**이다.

**여전히 중요한 것 (SEO와 겹침):**
- 크롤링 가능성 (robots.txt, sitemap.xml)
- E-E-A-T 시그널 (전문성, 권위, 신뢰)
- 콘텐츠 품질과 깊이
- 구조화 데이터 (Schema.org)
- 기술적 기본기 (페이지 속도, 모바일 대응, HTTPS)

**GEO에서 새로운 것:**
- **인용 가능성 (Quotability)** — 60~100단어 단위로 독립적으로 추출 가능한 문단
- **다중 출처 검증** — AI는 여러 출처에서 교차 확인. 고유한 데이터 포인트가 중요
- **답변 우선 포맷** — 첫 40~60단어에 핵심 답변을 배치
- **llms.txt** — AI 모델을 위한 새로운 파일 표준 (robots.txt의 AI 버전)
